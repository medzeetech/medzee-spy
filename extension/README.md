# Medzee Spy — Chrome Extension (MV3)

Chrome extension that reads WhatsApp Web (via `@wppconnect/wa-js`) and forwards
chat data to the Medzee Spy backend for commercial analysis.

Status: **scaffold only** (Wave 4 T10). Source files are stubs filled in by
subsequent tasks (T11–T16) in `.specs/features/f8-chrome-extension-ingestion/tasks.md`.

## Stack

- **Manifest V3** (Chrome 102+)
- **TypeScript 5.6** strict mode
- **Vite 6** + **@crxjs/vite-plugin** for MV3 manifest handling and HMR
- **React 19** (popup UI)
- **@wppconnect/wa-js** (WhatsApp page-world API)
- **sharp** for icon rasterization

### Build approach: CRXJS

We use `@crxjs/vite-plugin` (beta) because it is the well-trodden path for MV3
+ Vite: it rewrites `src/...` paths in `manifest.json` to compiled JS, bundles
content scripts as IIFE, hoists `web_accessible_resources`, and gives HMR for
the popup during `npm run dev`.

If CRXJS becomes brittle on this stack, the fallback is a manual multi-entry
Vite config:

```ts
build: {
  rollupOptions: {
    input: {
      'service-worker': 'src/service-worker.ts',
      'probe': 'src/content-scripts/probe.ts',
      'collector': 'src/content-scripts/collector.ts',
      'wa-collector': 'src/page-world/wa-collector.ts',
      'popup': 'src/popup/popup.html',
    },
    output: { entryFileNames: '[name].js', chunkFileNames: 'chunks/[name]-[hash].js' },
  }
}
```

…paired with a post-build script that rewrites `manifest.json` paths.

Icons are rasterized from `../frontend/src/assets/logo-medzee-spy.svg` via
`sharp` at build time and are **gitignored**. Run `npm run build:icons` to
regenerate (idempotent — skips if PNGs already exist).

## Install / development

```bash
cd extension
npm install
npm run build
```

Outputs to `dist/`. To side-load in Chrome:

1. Open `chrome://extensions`
2. Enable **Developer mode** (top-right)
3. Click **Load unpacked**
4. Select the `extension/dist/` folder
5. The Medzee Spy icon should appear in the toolbar

### Watch mode

```bash
npm run dev
```

Vite rebuilds on save; reload the extension from `chrome://extensions` to pick
up changes (CRXJS provides popup HMR but content/background scripts still need
a manual reload).

### Type-checking

```bash
npm run typecheck
```

## Notes

- `node_modules/`, `dist/`, and `public/icons/` are gitignored.
- The extension talks to `https://*.medzee.com/*` in prod and
  `http://localhost:5173/*` + `http://localhost:8000/*` in dev (see
  `host_permissions` in `manifest.json`).
